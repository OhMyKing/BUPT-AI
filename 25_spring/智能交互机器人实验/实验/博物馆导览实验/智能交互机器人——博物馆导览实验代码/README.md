# 博物馆导览机器人

本项目实现了一个智能博物馆导览机器人系统，可在虚拟博物馆环境中自主导航、避障、面向展品并通过手臂指向展品，同时提供自然语言和视觉多模态交互功能。

## 功能特点

- **智能路径规划**：使用Dijkstra算法实现从当前位置到目标展品的最优路径规划
- **多级避障系统**：基于深度相机的实时避障，根据障碍物距离动态调整行走策略
- **展品指向功能**：精确面向展品并使用手臂进行指向动作
- **多模态交互**：结合Qwen2.5-VL大语言模型，支持自然语言和视觉输入的智能交互
- **模块化设计**：系统各功能模块间接口明确，便于扩展和维护

## 系统要求

- **操作系统**：Ubuntu 20.04
- **ROS版本**：ROS Noetic
- **仿真平台**：CoppeliaSim
- **编程语言**：Python 3.8
- **机器人平台**：Roban仿人机器人

## 使用方法

### 1. 启动系统

只需运行主程序，它会自动启动所有必要的组件：

```bash
python3 main.py
```

此命令将：

- 启动ROS核心
- 启动CoppeliaSim仿真环境并加载博物馆场景
- 启动必要的ROS节点
- 启动导览和介绍功能

### 2. 导航操作

系统启动后，在终端中会提示输入目标位置编号(1-8)：

```
请输入目的地编号(1-8)，或输入q退出:
```

输入数字后，机器人将自动规划路径并导航到指定展位。

### 3. 展品交互

到达展品位置后，机器人会自动面向展品并使用手臂指向展品。在另一个终端窗口中，您可以通过自然语言与机器人交互，询问关于展品的信息：

```
请输入您的问题: 这个展品是什么？
```

机器人会结合视觉信息和您的问题提供相关回答。

### 4. 退出系统

在导航终端中输入`q`可退出导航系统，在主程序终端中按`Ctrl+C`然后输入`q`可完全退出系统。

## 核心模块说明

### 1. 导航系统 (guide.py)

- **MuseumGuideRobot类**：主控制类，管理机器人导航和避障
- **build_path方法**：使用Dijkstra算法计算最短路径
- **navigate_to_destination方法**：控制机器人到达指定位置
- **move方法**：基于障碍物距离调整机器人行走

### 2. 逆运动学控制 (IKController.py)

- **IKController类**：管理机器人手臂动作
- **raise_hand方法**：实现手臂指向展品的动作

### 3. 多模态交互 (introduce.py)

- **MuseumGuideRobot类**：处理用户问题和相机图像
- **process_user_input方法**：处理用户输入并获取LLM响应
- **create_messages方法**：创建API请求消息

## 高级配置

### 自定义展品信息

在`guide.py`文件中，可以修改`EXHIBIT_INFO`字典来更新展品描述：

```python
EXHIBIT_INFO = {
    2: """
=== 展品#1: 古代石器 ===
这件展品是距今约8000年前的新石器时代石斧。
...
""",
    # 添加更多展品信息
}
```

### 调整导航参数

在`guide.py`文件中，可以修改以下常量来调整导航行为：

```python
GAIT_RANGE = 0.05  # 行走步长(米)
ROTATION_RANGE = 10.0  # 旋转角度(度)
DESTINATION_THRESHOLD = 0.2  # 到达目的地的距离阈值(米)
```

## 常见问题

1. **问题**: 机器人无法正确启动 **解决方案**: 确保ROS环境已正确设置，运行`source /opt/ros/noetic/setup.bash`
2. **问题**: 无法连接到CoppeliaSim **解决方案**: 确保CoppeliaSim已正确安装且可执行权限设置正确
3. **问题**: 导航过程中机器人卡住 **解决方案**: 按`q`退出当前导航，然后重新输入目标位置
