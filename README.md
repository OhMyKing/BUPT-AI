# BUPT-AI

不同于北邮其他专业，人工智能专业建立晚、课程新、材料少，课程信息的差异随之成为一种壁垒。作为这一专业2026届的本科毕业生，一路走来，诸多不便，仍在心头。因而建立本仓库，希望能够广泛收集本专业的课程材料，帮助后来者少一点壁障，节约一些时间，投入到更有趣的事物中去。

## 说明

北京邮电大学人工智能专业本科课程资料整理仓库，覆盖作业、实验、课设、课件、考题、复习资料与毕业设计材料。

资料按 `学期/课程/资源类型` 组织。学期目录使用完整英文季节名，例如 `24_fall` 为 2024 年秋季学期，`25_spring` 为 2025 年春季学期。

## 使用说明

- 资料仅供学习、复习与课程交流，请遵守学校规定和课程学术诚信要求。
- 考题具有时效性，课程安排与考核方式请以任课教师当学期要求为准。
- 代码项目中的模型权重、公开数据集、依赖目录和真实 API 密钥均已移除；运行前请阅读项目 README。
- 演示视频、答辩 PPT 和项目报告作为成果说明保留。大型媒体应使用 Git LFS 管理。
- 教材、论文、课件等材料的著作权归原作者或出版机构所有。如有侵权，请提交 Issue 联系移除。

## 参与贡献

提交新资料前请阅读 [贡献与 PR 规范](docs/CONTRIBUTING.md)。请勿提交真实凭证、个人敏感材料、公开数据集、训练模型、依赖目录或编译产物。

### 使用 Agent 辅助整理

你可以让 Codex、Claude Code、OpenClaw、Hermes 等具备本地文件和 Git 操作能力的 Agent 阅读 [Agent 整理协议](CLAUDE.md)，辅助完成资料盘点、课程归类、安全清理、README 补充、Git LFS 配置以及 PR 准备。通常只需向 Agent 提供待整理文件或文件夹的绝对路径；Agent 会先生成整理方案，在涉及复制整理、隐私处理、大文件、提交和 PR 等节点向你提审确认。

可直接复制下面的提示词，并替换路径：

```text
请先阅读当前 BUPT-AI 仓库根目录的 CLAUDE.md、README.md 和
docs/CONTRIBUTING.md，并严格遵循其中的工作流。

请整理以下本地课程资料：
- /绝对路径/课程资料文件夹
- /绝对路径/另一份资料.pdf

你需要递归盘点文件，判断学期、课程和资源类型，检查重复版本、凭证、
个人敏感信息、模型、公开数据集、构建产物及损坏文件，并规划它们在
BUPT-AI 中的位置。不要修改或删除源文件。

先向我提交完整整理方案，得到确认后再复制和修改；执行完成后再次向我
展示差异、安全检查结果及大文件清单，得到确认后才能 commit。只有我明确
同意时才可以 push 和创建 PR。

对于大文件，请检查 GitHub 普通文件限制和仓库 Git LFS 配额：演示视频及
确需保留的大型二进制资料使用 Git LFS；训练模型和公开数据集不要提交，
改在项目 README 中说明官方获取方式。如果文件已经错误进入 Git 历史，
先说明影响和迁移方案，得到确认后才能改写历史。
```

Agent 可以减少机械整理工作，但不能替代贡献者对版权、隐私、学术诚信和最终变更的人工判断。扫描型 PDF、图片水印、视频画面与音轨尤其需要人工复核。

## 留言墙

除了课程资料，这个仓库也希望留下学习者之间的经验与温度。参与 PR 的贡献者可以前往 [留言墙](docs/messages/README.md)，在 `docs/messages/` 下写下想对学弟学妹说的话，例如课程建议、学习心得、踩坑记录或一段简单的鼓励。

留言可以使用姓名、用户名或昵称命名，并通过独立 Markdown 文件提交。具体目录结构和建议格式见留言墙说明。
